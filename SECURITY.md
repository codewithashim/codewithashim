# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly by emailing [codewithashim@gmail.com](mailto:codewithashim@gmail.com) with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

**Please do not open public issues for security vulnerabilities.**

## Response Timeline

- **Initial Response:** 24-48 hours
- **Assessment & Fix:** 1-2 weeks (depending on severity)
- **Disclosure:** After a fix is available and deployed

## Security Best Practices

When using this project:

- Keep dependencies up to date
- Use environment variables for sensitive data (never commit secrets)
- Follow OWASP guidelines for web application security
- Review code changes before deploying to production
- Enable CI/CD security scanning

## Supported Versions

Security updates are provided for:

- Latest major version
- Previous major version (limited support)

Older versions are not supported and may contain unfixed vulnerabilities.

## Security Updates

We follow semantic versioning. Security patches are released as patch versions (e.g., 1.0.1).
